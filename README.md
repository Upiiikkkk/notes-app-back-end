1. server.js - Main Server Configuration
Purpose: Initializes and starts the Hapi.js server with CORS configuration.

Key Components:
javascript
const server = Hapi.server({
    port: 3000,
    host: 'localhost',
    routes: {
        cors: {
            origin: ['*'],  // Allows requests from any origin
        },
    },
});
Functions:
init(): Starts the server on port 3000

Configures CORS for cross-origin requests

Registers all routes from routes.js

2. routes.js - API Route Definitions
Purpose: Maps HTTP methods and paths to their corresponding handler functions.

API Endpoints:
Method	Path	Handler	Description
POST	/notes	addNoteHandler	Create a new note
GET	/notes	getAllNoteHandler	Get all notes
GET	/notes/{id}	getByIdHandler	Get note by ID
PUT	/notes/{id}	editByIdHandler	Update note by ID
DELETE	/notes/{id}	deleteByIdHandler	Delete note by ID
3. handler.js - Request Handlers
Purpose: Contains all business logic for processing API requests.

Available Handlers:
a. addNoteHandler(request, h)
Creates a new note

Request Body:

json
{
    "title": "Note Title",
    "tags": ["tag1", "tag2"],
    "body": "Note content"
}
Response:

Success (201):

json
{
    "error": false,
    "status": "success",
    "message": "Catatan berhasil ditambahkan",
    "data": {
        "notes": "generated-id"
    }
}
Error (500):

json
{
    "status": "fail",
    "message": "Catatan gagal ditambahkan"
}
b. getAllNoteHandler()
Retrieves all notes

Response:

json
{
    "status": "success",
    "data": {
        "notes": [...]
    }
}
c. getByIdHandler(request, h)
Retrieves a specific note by ID

URL Parameter: id - Note identifier

Response:

Success (200):

json
{
    "status": "success",
    "data": {
        "note": {...}
    }
}
Error (404):

json
{
    "status": "fail",
    "message": "Catatan tidak ditemukan"
}
d. editByIdHandler(request, h)
Updates an existing note

URL Parameter: id - Note identifier

Request Body:

json
{
    "title": "Updated Title",
    "tags": ["updated", "tags"],
    "body": "Updated content"
}
Response:

Success (200):

json
{
    "status": "success",
    "message": "Catatan berhasil diperbarui"
}
Error (404):

json
{
    "status": "fail",
    "message": "Gagal memperbarui catatan, Id tidak ditemukan"
}
e. deleteByIdHandler(request, h)
Deletes a note by ID

URL Parameter: id - Note identifier

Response:

Success (200):

json
{
    "status": "success",
    "message": "Catatan berhasil dihapus"
}
Error (404):

json
{
    "status": "fail",
    "message": "Gagal menghapus catatan, Id tidak ditemukan"
}
4. notes.js - Data Storage
Purpose: Serves as an in-memory database for storing notes.

Structure:
javascript
const notes = [];  // Array to store all notes
module.exports = notes;
Note: This is a temporary in-memory storage. All data will be lost when the server restarts.

Each note in the system has the following structure:

javascript
{
    id: "unique-16-character-id",        // Generated by nanoid
    title: "Note Title",                 // User-provided
    tags: ["tag1", "tag2"],             // Array of tags
    body: "Note content",               // Main content
    createdAt: "2023-01-01T00:00:00Z",  // ISO timestamp
    updatedAt: "2023-01-01T00:00:00Z"   // ISO timestamp
}
üöÄ Quick Start Guide
Install Dependencies:

bash
npm init -y
npm install @hapi/hapi nanoid
Start the Server:

bash
node server.js
Test the API:

bash
# Create a note
curl -X POST http://localhost:3000/notes \
     -H "Content-Type: application/json" \
     -d '{"title":"My Note","tags":["test"],"body":"Content here"}'

# Get all notes
curl http://localhost:3000/notes

# Get specific note
curl http://localhost:3000/notes/{note-id}

# Update note
curl -X PUT http://localhost:3000/notes/{note-id} \
     -H "Content-Type: application/json" \
     -d '{"title":"Updated Title","tags":["updated"],"body":"New content"}'

# Delete note
curl -X DELETE http://localhost:3000/notes/{note-id}
